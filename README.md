# PolyLogos - 多模型 AI 对比与辩论工具

一个让多个大语言模型同时回答同一问题，或进行辩论对抗的 Web 工具。

## 项目地址

- 对比模式：https://ygleam1206.github.io/PolyLogos/
- 辩论模式：https://ygleam1206.github.io/PolyLogos/debate.html

## 功能列表

### 一、对比模式

- 支持同时向多个模型提问，并排显示回答
- 流式输出：实时显示模型生成内容
- 一键复制任意模型的回答
- 显示回答字符数，便于对比详略
- 导出对比结果为图片

### 二、辩论模式

选择两个模型分别担任正反方，围绕辩题进行多轮辩论，最后由 AI 裁判打分并宣布胜者。

#### 辩论流程

1. 用户输入辩题，选择正方模型和反方模型
2. 设置辩论轮数（2 轮 / 3 轮 / 4 轮）
3. 正方先发言，阐述支持立场
4. 反方针对正方观点进行反驳
5. 交替发言，完成设定轮数
6. AI 裁判对双方进行打分，输出胜负判定

#### 裁判评分维度

| 维度 | 说明 | 分值范围 |
|------|------|----------|
| 论点深度 | 观点是否有洞察力，是否切中核心 | 1-10 |
| 论据质量 | 是否有具体例子、数据或逻辑支撑 | 1-10 |
| 逻辑严密性 | 推理是否连贯，有无明显漏洞 | 1-10 |
| 反驳有效性 | 是否有效回应对方观点 | 1-10 |
| 表达清晰度 | 语言是否清晰、有条理 | 1-10 |

#### 裁判输出内容

- 正方/反方各维度得分及总分
- 明确胜者（正方或反方）
- 三条以上获胜理由
- 双方优势分析
- 一句话总结

#### 支持的辩论模型

| 模型 | 说明 |
|------|------|
| DeepSeek | 需配置 API Key |
| 通义千问 | 需配置 API Key |
| 智谱 GLM | 完全免费，推荐 |
| Kimi | 需配置 API Key |
| ChatGPT | 需配置 API Key |
| Gemini | 需配置 API Key |

裁判模型自动选择：优先使用智谱 GLM，其次通义千问，最后 DeepSeek。

### 三、标签分类功能

对历史对话进行分类管理，便于检索和筛选。

#### 系统预设标签

| 标签名称 | 适用场景 |
|----------|----------|
| 编程 | 代码、技术、开发相关的问题 |
| 生活 | 日常生活、消费、娱乐相关的问题 |
| 学习 | 学习方法、知识科普、教育相关的问题 |
| 创意 | 写作、设计、头脑风暴相关的问题 |
| 其他 | 无法归类的通用问题 |
| 未分类 | 尚未标注的对话（默认） |

#### 标签管理操作

- **自动分类**：每次对话保存时，AI 自动判断问题类别并打上对应标签
- **手动修改**：用户可以在历史记录中为任意对话修改标签
- **创建自定义标签**：用户可以创建自己的标签（如「工作」、「美食」、「旅行」）
- **删除标签**：系统预设标签不可删除，自定义标签可删除
- **标签颜色**：每个标签有独立的颜色标识，便于视觉区分

#### 标签筛选功能

- 点击任意标签，历史记录列表只显示该标签下的对话
- 支持「全部」标签，查看所有记录
- 筛选结果实时更新，不影响数据存储

#### 标签与搜索的联动

当同时使用关键词搜索和标签筛选时，结果取交集（既匹配关键词又属于选中标签）。

### 四、智能整合

- 将多个模型的回答融合成一份完整答案
- 自动去重，标记冲突观点
- 标注每条信息的来源模型

### 五、模型管理

- 启用/禁用任意模型
- 拖拽调整模型显示顺序
- 按响应速度排序
- API Key 统一配置，保存到本地

### 六、历史记录

- 自动保存所有对话（问题 + 所有模型的回答）
- 支持全文搜索（搜索范围：问题 + 回答内容）
- 搜索关键词高亮显示
- 单条删除 / 清空全部

## 技术栈

- 前端：HTML + Tailwind CSS + JavaScript
- API 调用：直接调用各模型 HTTP 接口
- 存储：localStorage（配置、历史记录、标签）

## 使用说明

1. 首次使用时，点击页面顶部的「模型管理」
2. 开启需要使用的模型，填写对应的 API Key
3. 在对比模式中输入问题，点击发送
4. 在辩论模式中选择正反方模型和辩题，点击开始辩论
5. 历史记录会自动保存，可使用标签或搜索筛选
